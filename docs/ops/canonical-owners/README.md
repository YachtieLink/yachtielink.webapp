# Canonical Owners

These docs define the intended source of truth for the areas where YachtieLink has repeated drift.

Use them during planning, build reviews, and cleanup work to answer:

- where should new logic go?
- which helper or read model owns this rule?
- which legacy surfaces should not gain new behavior?

Current focus:

- `cv-onboarding.md`
- `profile-read-models.md`
- `media-and-pro-gating.md`

If a branch adds a second live owner for one of these areas, treat that as an architecture finding, not just a style concern.
