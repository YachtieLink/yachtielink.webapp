# UX/IA Audit — All 5 Tabs
**Date:** 2026-04-02 | **Rally:** 009 Pre-MVP Polish

---

## The Big Problems (Cross-Tab)

These are systemic issues that no single tab can fix alone.

### 1. Nobody knows where to edit their identity

A user wants to change their name. They're on "My Profile" — the obvious place. But there's no edit button on the hero card. Name/handle/role editing lives on the More tab, behind "Edit profile & contact info," which links to a separate full-form page. A user would never look there first. **The most fundamental action — changing who you are — can't be done from the tab named after you.**

### 2. Profile editing has two doors to the same room

The Profile tab has "Edit" links on every section card. The More tab has "Edit profile & contact info" which goes to a mega form covering the same fields plus more. Two paths, no signal which is primary. A user who finds one won't know the other exists. A user looking might try the wrong one first.

**Fix:** Remove "Edit profile & contact info" from More entirely. Profile tab inline edits are the primary path. Add a tap-to-edit affordance on the hero card for name/role/handle.

### 3. Things are where the system puts them, not where users look

| Item | Currently | Where a user would look |
|------|-----------|------------------------|
| Edit name/role | More tab | Profile tab |
| Saved Profiles | More tab | Network tab |
| Cert Documents | Insights tab | Profile (certs section) or CV tab |
| Subscription management | Insights tab AND More tab | More/Settings only |
| Yacht management | Network tab (Yachts) | CV tab (it's employment history) |
| CV Details (smoking, tattoos) | CV tab | Profile tab (personal attributes) |

The information architecture follows database structure, not user intent.

### 4. The toggles on Profile don't explain themselves

The 2-column section grid has toggles on every card. These control public visibility — but a first-time user thinks "on/off" means "enable/disable." The instructional text "Choose what shows on your public profile" sits above the grid where nobody reads it. The concept of "private profile vs. public profile" is never established. Users don't know they have two profiles.

### 5. The "Endorse" button means the opposite of what it says

On the Colleagues tab, each colleague has an "Endorse" button. A user reads this as "write an endorsement for this person." It actually means "request an endorsement FROM this person." The button label is backwards. This will cause confusion and embarrassment.

### 6. Free users get a dead tab

The Insights tab for free users is five locked cards and an upgrade CTA. Zero useful information. A top-level nav tab that shows nothing but a paywall will be tapped once and abandoned. That's prime navigation real estate wasted on a billboard.

### 7. Three tabs show the same yacht data differently

Yachts appear on: Profile (experience section), CV (generated document), Network (Yachts tab with management). A user adding a yacht on the Network tab is actually creating an employment record that shows on Profile and CV. But nothing communicates this connection. A user might add the same yacht twice thinking these are different systems.

## Per-Tab Findings

### My Profile

**Identity crisis:** Tries to be a profile preview, a section manager, AND a coaching tool — all interleaved vertically with no separation.

| Issue | Severity | Fix |
|-------|----------|-----|
| Can't edit name/role/handle from here | Critical | Add edit affordance on hero card |
| Profile Strength buried below fold | High | Move to right after hero card |
| Visibility toggles misunderstood as on/off | High | Add inline context ("visible on public profile") or rethink the toggle pattern |
| 2-column grid is a wall of identical cards | Medium | No visual hierarchy — Experience and Certs should outweigh Hobbies |
| Photos has two entry points (top strip + grid card) | Medium | Remove grid card, keep top strip |
| No preview of assembled public profile | Medium | Add "Preview" link near the top |
| Bio not visible on own profile page | Medium | Show bio inline, not just behind Edit |
| Cert expiry buried in a grid card | Medium | Surface as alert/notification if expiring |
| No inline bio display | Low | Show the bio text, not just a summary |

### CV

**Identity crisis:** Conflates data input (CV Details, import), document management (generate/download), and sharing settings.

| Issue | Severity | Fix |
|-------|----------|-----|
| "Update from new CV" could overwrite edited profile data with no warning | Critical | Add confirmation: "This will re-parse and may overwrite edits you've made" |
| No link back to edit source data | High | Add "Edit your experience on Profile" link |
| "Visitor Downloads" label is platform jargon | High | Rename to "Sharing settings" or "Who can download" |
| CV Details (smoking, tattoos, DL) are personal attributes, not CV settings | Medium | Move to Profile (already decided) |
| Template picker gets equal visual weight to primary actions | Medium | Collapse to secondary setting |
| No inline CV preview | Medium | Show thumbnail or first-page snapshot |
| No share link / QR code on the CV page itself | Medium | Add copy-link button |
| If CV becomes output-only, is it worth a whole tab? | Question | Monitor usage — may fold into Profile eventually |

### Insights

**Identity crisis:** Simultaneously analytics dashboard, cert management shortcut, subscription panel, and (for free users) pure sales page.

| Issue | Severity | Fix |
|-------|----------|-----|
| Free users see zero useful content — just a paywall | High | Show at least one real metric (total profile views lifetime) for free |
| Cert Document Manager doesn't belong here | High | Move to Profile certs section or More (already decided) |
| Subscription card duplicated here AND on More | Medium | Remove from Insights (already decided) |
| No trend indicators — numbers without context | Medium | Add "up/down X% vs last period" |
| "Insights" overpromises if it's just three counters | Medium | Either add actionable recommendations or rename |
| Time range selector triggers full page reload | Low | Consider client-side filtering for snappiness |

### Network

**Identity crisis:** Doing the job of four features: endorsement management, colleague browsing, yacht management, yacht discovery.

| Issue | Severity | Fix |
|-------|----------|-----|
| "Endorse" button means "request endorsement from" — backwards | Critical | Rename to "Request" or "Ask for endorsement" |
| No people search on Colleagues tab | High | Add search/filter |
| Endorsements tab shows empty state with no education about what endorsements are | High | Empty state should explain value, show examples |
| Yacht management is really CV/employment data surfaced redundantly | Medium | Unified view (already decided) addresses most of this |
| Saved Profiles buried on More tab, URL is /network/saved | Medium | Move entry point to Network tab |
| "Requests received" (endorse others) mixed with "my endorsements" | Medium | Separate clearly: "Write endorsements" vs "My endorsements" |
| No way to proactively write an endorsement for someone | Medium | Consider allowing unprompted endorsements |
| My Yachts duplicates Profile experience list | Low | Unified view resolves this |

### More / Settings

**Identity crisis:** Labeled "More," titled "Settings," contains profile editing, people management, account config, and product info.

| Issue | Severity | Fix |
|-------|----------|-----|
| "Edit profile" here duplicates Profile tab's edit links | High | Remove from More entirely |
| Saved Profiles here instead of Network | High | Move to Network |
| Page titled "Settings" but tab says "More" | Medium | Pick one — recommend "Settings" |
| Delete account too close to Download data | Medium | Move to very bottom, visual separation |
| "Download my data" is ambiguous | Medium | Rename to "Export personal data (GDPR)" |
| 7 groups for 12 items — too many groups | Medium | Collapse to 3-4 groups |
| Feature Roadmap under HELP is unexpected | Medium | Make standalone or surface contextually |
| Send feedback goes to email | Medium | Replace with in-app form |
| Sign out button is visually louder than everything else | Low | Demote to text link |
| Missing: notification prefs, email prefs, blocked users | Low | Add as app grows |

## Proposed Ownership Map

What each tab should own — and ONLY own:

| Tab | Owns | Does NOT own |
|-----|------|-------------|
| **My Profile** | Everything a captain/recruiter sees: name, role, photos, bio, experience, certs, endorsements received, skills, visibility toggles. All content editing. Profile strength coaching. | Account settings, billing, analytics |
| **CV** | The document: generate, preview, download, share, template. Output-only. | Data entry (experience editing), personal attributes (smoking/tattoos), cert management |
| **Insights** | Analytics only: views, downloads, shares, saves, who viewed, trends. Pro gets detail, free gets at least one real number. | Cert documents, subscription management, anything non-analytics |
| **Network** | People: colleagues, endorsements (display + management), saved profiles, yacht-grouped connections, invite flow. | Yacht management as CV data, global yacht search |
| **Settings** | Account: login, billing, appearance, notifications, data export, delete. App-level config. Legal. Support. | Profile editing, saved profiles, roadmap (unless contextual) |

---

## Top 5 Actions (Priority Order)

1. **Add edit affordance on Profile hero card** for name/role/handle. Remove "Edit profile" from More tab.
2. **Rename "Endorse" buttons to "Request"** on Colleagues tab. This is actively misleading users.
3. **Give free users one real metric on Insights** so the tab isn't a dead end.
4. **Move Saved Profiles from More to Network.**
5. **Add confirmation warning on "Update from new CV"** — it can silently overwrite edited profile data.
