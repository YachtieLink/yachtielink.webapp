# Icon System Audit — Action Icons Across Platform

## Problem

As features grow, action buttons are getting crowded (Request, Endorse, Invite, Report, Save, Share, Transfer). Text labels take too much space, especially on mobile at 375px. Need a consistent icon language so users learn the icons once and recognize them everywhere.

## Vision

Define a standard icon for every common action. Use icon-only buttons where space is tight (colleague rows, cards), icon + label where there's room (pages, modals). Consistent across all entry points.

## Needs /grill-me

- Which actions get icons vs text vs both?
- Which lucide icons map to which actions?
- When to use icon-only vs icon+label (breakpoint rules?)
- Tooltip on icon-only buttons?
- Color coding — should action type influence icon color?

## Notes

- Lucide is already the icon library in use
- This should feed into the design system docs (`docs/design-system/style-guide.md`)
