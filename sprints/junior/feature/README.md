# Feature Sprints

For quick feature additions or tweaks that weren't in the main sprint plan but need shipping now. These are real product work — just reactive or fast-tracked.

Keep them tight. If a feature sprint starts growing, it probably belongs in the next major sprint instead.

## Sprint Template

Create a subfolder like `feature-<short-slug>/` and add a `README.md`:

```markdown
# Feature: [Short description]

**Started:** YYYY-MM-DD
**Status:** ⚡ In Progress / ✅ Shipped
**Priority:** High / Medium / Low

## Why Now
Why this couldn't wait for the next major sprint.

## Scope
What this adds or changes. Keep it tight.

## Out of Scope
What this does NOT touch.

## Files Affected
- ...

## Exit Criteria
How do we know it's done and working?
```
