# UI/UX Sprints

For layout, styling, responsive fixes, animation polish, and visual improvements. Not new features — making existing things look and feel better.

These often happen between major sprints or when something looks off and you want to fix it properly.

## Sprint Template

Create a subfolder like `ui-<short-slug>/` and add a `README.md`:

```markdown
# UI/UX: [Short description]

**Started:** YYYY-MM-DD
**Status:** 🎨 In Progress / ✅ Done
**Affects:** [pages / components]

## What Looks Wrong / What Could Be Better
Describe the problem or the improvement opportunity.

## Approach
What we're changing and why.

## Screens / Components Affected
- ...

## Before / After
(Optional — paste screenshots or describe the change)

## Exit Criteria
What does "done" look like?
```
