## Key Decisions

- **CV parse root cause:** Not a bad import — `PDFParse` named export is correct in v2. Root cause was Next.js bundler breaking the dynamic import. Fix: `serverExternalPackages: ['pdf-parse']`
- **Photo downgrade behaviour:** On Pro→Free downgrade, extra photos are hidden (not deleted). User sees first 3 with upgrade notice. Re-upgrading instantly restores all photos.
- **isPro default:** Changed from `false` to `true` so plan lookup failures don't penalise paid users. Server enforces real limits on upload.
- **Button margin approach:** Used `max(env(safe-area-inset-top, 0px), 1rem)` instead of just increasing the fallback — this guarantees at least 1rem even on devices that report 0 for safe-area.
