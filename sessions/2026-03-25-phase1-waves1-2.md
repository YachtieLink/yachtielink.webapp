## Decisions Made

- **dob REVOKE workaround:** Server-side age computation via authenticated query. Age integer passed to client, raw DOB never serialized. Only shown to logged-in viewers.
- **cv_public null semantics:** null = public (matching CvActions default). Explicit `=== false` check everywhere.
- **Profile query architecture:** Shared helpers in `lib/queries/profile.ts` rather than inline Supabase calls. Types in separate `types.ts` file. Tradeoff: profile.ts grew to 446 LOC — split planned at Wave 5.
- **Test backlog as pre-commit requirement:** Canonical doc in `docs/ops/`, wired into AGENTS.md. Agents must add items; only founder checks them off.

## Gotchas

- Supabase FK joins return array types in TypeScript but single objects at runtime → need `as unknown as` casts at boundary
- `started_at` on attachments can be null (imported CVs without dates) — `profile-summaries.ts` needed null guard
- Git push to main blocked by branch protection — must use feature branches + PRs
